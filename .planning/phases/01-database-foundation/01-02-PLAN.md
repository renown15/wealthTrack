Create `backend/app/models/reference_data.py` with four lookup table models.

Each model follows the same structure (from RESEARCH.md Pattern 2):
- id: Mapped[int] - primary key
- code: Mapped[str] - unique identifier (e.g., 'CASH_ISA'), indexed
- name: Mapped[str] - display name (e.g., 'Cash ISA')
- description: Mapped[str | None] - optional description
- sort_order: Mapped[int] - for UI ordering (default 0)
- is_active: Mapped[bool] - soft delete flag (default True)
- created_at: Mapped[datetime] - timestamp

Models to create:

1. **AccountType** - `__tablename__ = "account_types"`
   - For: Savings, Current, Cash ISA, Stocks & Shares ISA, Lifetime ISA, SIPP, Premium Bonds

2. **AccountStatus** - `__tablename__ = "account_statuses"`
   - For: Active, Closed, Dormant, Pending

3. **EventType** - `__tablename__ = "event_types"`
   - For: Balance Update, Interest Payment, Dividend, Deposit, Withdrawal

4. **CredentialType** - `__tablename__ = "credential_types"`
   - For: Username, Password, PIN, Security Question, Memorable Word

Use SQLAlchemy 2.0 style with Mapped[] type annotations:
```python
from datetime import datetime
from sqlalchemy import String, Integer, Boolean, DateTime
from sqlalchemy.orm import Mapped, mapped_column
from app.database import Base

class AccountType(Base):
    """UK-specific account types lookup table."""
    __tablename__ = "account_types"

    id: Mapped[int] = mapped_column(primary_key=True)
    code: Mapped[str] = mapped_column(String(50), unique=True, nullable=False, index=True)
    name: Mapped[str] = mapped_column(String(100), nullable=False)
    description: Mapped[str | None] = mapped_column(String(255))
    sort_order: Mapped[int] = mapped_column(Integer, default=0, nullable=False)
    is_active: Mapped[bool] = mapped_column(Boolean, default=True, nullable=False)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow, nullable=False)

    def __repr__(self) -> str:
        return f"<AccountType(code={self.code}, name={self.name})>"
```
