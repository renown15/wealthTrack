# WealthTrack Application Specification## Project OverviewWealthTrack is a strategic wealth management web application. It must prioritize **maintainability, testability, and scalability** from day one.**Tech Stack:**- Front-end: MVC JavaScript/TypeScript- Back-end: Python (FastAPI recommended)- Database: PostgreSQL- Deployment: Dockerized environment- CI/CD: Automated linting, type checking, testing, and deployment## Core Requirements### 1. File Structure & Size- Max **200 lines per file**- Clear modular structure:/models/views/controllers/services/tests- Isolated modules for maintainability### 2. Testing & Coverage- Unit tests for **Python** and **JS/TS**- Test coverage: **³90%**- Frameworks:- Python: `pytest`- JS/TS: `Jest` or `Vitest`### 3. Type Safety & Linting- Python: `mypy`- JS/TS: `tsc` + ESLint (strict)- Enforce in CI pipeline### 4. Docker & Deployment- Dockerize front-end, back-end, and database- `docker-compose.yml` for local development- CI/CD pipeline: lint ? type check ? tests ? Docker build ? deploy### 5. MVC Architecture- **Models:** DB schemas, validation- **Views:** Templates (browser) or API endpoints (mobile/web)- **Controllers:** Orchestrate models and views- **Services:** Business logic### 6. Cross-Platform Support- Mobile-first, responsive UI- API to support both browser and mobile clients### 7. CI/CD- GitHub Actions workflow:- Lint (Python & TS)- Type check- Run tests with coverage- Build & deploy Docker images## Instructions for Copilot1. Scaffold project with folders and modular files (**²200 lines each**)2. Implement **sample feature: user registration** - API endpoints (Python FastAPI) - Front-end pages (JS/TS MVC) - Models, controllers, services - Unit tests for each layer3. Include Docker setup: - Dockerfiles for front-end and back-end - `docker-compose.yml` linking all services4. Include CI/CD workflow: - GitHub Actions YAML - Steps: lint ? type check ? test ? Docker build5. Enforce type hints, ESLint, and `mypy`6. Ensure modular, maintainable, scalable design## Deliverables- Complete project scaffold- Sample registration feature implemented- Unit tests and type checks- Dockerfiles and `docker-compose.yml`- CI/CD workflow file- ESLint and `mypy` configurations- README with instructions for running locally and deploying## Notes- Prioritize code quality, modularity, and maintainability- Mobile & browser support from day one- Keep files ²200 lines- Strategic design for future growth
