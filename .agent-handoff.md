# Database Sync Debugging - Handoff to Better Agent

## Summary
I've been trying to create a `make sync-db-to-pi` target that dumps the local database and restores it to a Raspberry Pi. I've made a complete mess of it and need help.

## What Should Happen
1. Dump the local postgres database (localhost:5433) with schema + data
2. Transfer gzipped dump to the Pi via SCP
3. Drop the existing database on the Pi
4. Create a fresh database
5. Restore the dump into the fresh DB
6. Run migrations to ensure schema is current
7. Clean up temp files

## What's Actually Happening
The schema gets created and migrations run fine, but **user profile data is not being restored**. 

### Current State
- **Local database (.env.dev)**: Has 2 UserProfile records (Mark Lewis, John Doe) with emails and passwords
- **Pi database after restore**: UserProfile table exists but is EMPTY (0 rows)
- **All other tables**: Same problem — schema present, data absent

The dump command works (tested manually), SCP works, table creation works, but the INSERT statements from the dump aren't executing on the Pi.

## What I've Tried
1. First attempt: Dump → restore without dropping existing DB → partial corrupt data
2. Second attempt: Dump → DROP DB → CREATE fresh DB → restore
3. Added `alembic upgrade head` to re-run migrations after restore

All fail at the same point: data doesn't come through.

## Current Makefile Target
```makefile
sync-db-to-pi:
	@if [ ! -f .env.dev ]; then echo "❌ .env.dev not found"; exit 1; fi
	@if [ ! -f .env.pi ]; then echo "❌ .env.pi not found"; exit 1; fi
	@. ./.env.dev && \
	echo "Dumping local database from localhost:$$DB_PORT..." && \
	pg_dump -h localhost -p $$DB_PORT -U $$DB_USER -d $$DB_NAME --no-password 2>/dev/null | gzip > /tmp/wealthtrack_dump.sql.gz && \
	echo "Dump complete: /tmp/wealthtrack_dump.sql.gz" && \
	echo "Transferring to $(PI_USER)@$(PI_HOST)..." && \
	scp /tmp/wealthtrack_dump.sql.gz $(PI_USER)@$(PI_HOST):/tmp/ && \
	echo "Dropping existing database on Pi..." && \
	ssh $(PI_USER)@$(PI_HOST) "cd ~/wealthTrack && docker compose --env-file .env.pi --profile prod exec -T db psql -U postgres -tc \"SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname='wealthtrack'\" && docker compose --env-file .env.pi --profile prod exec -T db psql -U postgres -tc \"DROP DATABASE IF EXISTS wealthtrack\" && docker compose --env-file .env.pi --profile prod exec -T db psql -U postgres -tc \"CREATE DATABASE wealthtrack OWNER wealthtrack\"" && \
	echo "Restoring database on Pi..." && \
	ssh $(PI_USER)@$(PI_HOST) "cd ~/wealthTrack && zcat /tmp/wealthtrack_dump.sql.gz | docker compose --env-file .env.pi --profile prod exec -T db psql -U wealthtrack -d wealthtrack" && \
	echo "Running migrations on Pi..." && \
	ssh $(PI_USER)@$(PI_HOST) "cd ~/wealthTrack && docker compose --env-file .env.pi --profile prod exec -T backend alembic upgrade head" && \
	ssh $(PI_USER)@$(PI_HOST) "rm /tmp/wealthtrack_dump.sql.gz" && \
	echo "✅ Database synced to Pi"
```
